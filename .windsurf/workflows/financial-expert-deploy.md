> 目标：**在部署前完成所有与本次改动相关的验证**，避免“先部署再发现问题”导致重复沟通和返工。

# 0. 原则（强制）

- 本次改动涉及到哪些模块/功能，就必须至少有一条**可复现的验证命令**覆盖它。
- **任何一条验证未通过：禁止部署**。
- 若验证依赖外部行情源（可能限流/超时），必须提供不依赖网络的确定性自测（本项目已提供 `scripts/selftest_trading_conditions.py`）。

# 1. 交易策略：买入/卖出条件（验收口径，必须与实现一致）

## 1.1 买入条件（Aggressive）

买入信号（`buy_price_aggressive_ok=true`）当且仅当同时满足：

- (Price > MA60)
- (MA60 ↑)：Slope% > 0
- (Price ≈ MA20)：|Price - MA20| / MA20 <= 2%
- (RSI Rebound)：RSI 昨日 < 前日 且 今日 > 昨日，且 昨日 < 40

买入条件描述输出字段：

- `buy_condition_desc`：一句话描述，包含：
  - “≈MA20”对应的价格区间（MA20±2%）
  - MA60、Slope%、RSI(前天/昨天/今天)
  - 最终是否“可以买入”或“不满足的子条件”

## 1.2 卖出条件

卖出信号（`sell_price_ok=true`）当满足以下任一：

- 止盈：
  - RSI>70 且顶背离（价格创新高但 RSI 未创新高；创新高阈值取近窗历史高点），或
  - 跌破 MA20
- 止损（口径已确认）：
  - **止损线 = 今天价格 - 2×ATR(14)**

卖出条件描述输出字段：

- `sell_condition_desc`：一句话描述，包含：
  - A：止盈条件（含 RSI 当前值、创新高阈值、MA20 阈值）
  - B：止损条件（含止损线数值：今日价-2×ATR）

# 2. 部署前验证（本地/开发机）

## 2.1 Python 语法检查（必做）

在项目根目录执行：

```bash
python3 -m py_compile api.py
```

## 2.2 交易策略条件描述自测门禁（必做）

该自测不依赖行情网络，验证：

- 买入描述中包含 MA20±2% 区间
- 卖出描述中包含止损线（今天价-2×ATR）

执行：

```bash
python3 scripts/selftest_trading_conditions.py
```

> 返回码非 0：禁止部署。

# 3. 部署（示例占位）

> 具体部署方式以项目现有脚本为准。部署前必须先通过第 2 节所有验证。

- 同步代码到服务器（如 rsync/scp 等）
- 服务端 `docker compose build && docker compose up -d`
- 健康检查（如有）

# 4. 常见排错

- 若前端页面不显示条件描述：确认后端返回包含 `buy_condition_desc/sell_condition_desc`，以及前端已更新类型定义并重新构建。
- 若行情源限流：部署前自测不依赖网络；线上运行指标接口可能受第三方数据源影响，属于运行时环境问题。
