# Agents V2 能力验证与 P0-P2 状态

## 本次已完成的修复（对齐 V1 体验）

### 1. 回复格式对齐 V1 ✅
- **实现**：`message-pipeline.js` 中 `formatReplyV1(res, userStoreLabel)` 按「小年：📊 标题 (门店 · 时间)」组回复。
- **数据审计/营运**：`agent-handlers.js` 中 data_auditor / ops_supervisor 返回 `reportTitle`、`timeLabel`，由 pipeline 拼成 V1 风格标题。
- **效果**：回复首行为「小年：📊 营业日报分析 (洪潮 · 昨日)」等形式，正文为列表与数据。

### 2. 数据源与门店/时间推断 ✅
- **门店**：`extractStoreFromText` 支持短名匹配（如「洪潮」「马己仙」→ 洪潮xxx店、马己仙xxx店），总部/未指定门店时从消息文本推断门店。
- **时间**：`extractTimeRangeFromText` 增加「最近/近期/近来」→ 近 7 天；`getTimeLabelChinese` 将时间范围转为「昨日」「上周」「本周」「最近7天」等中文标签并在标题和正文中使用。
- **无数据**：`validateEvidence` 允许含「无营业数据」「暂无数据」「0条」的回复，不再误判为证据不足；数据不足时提示「请提供门店（如：洪潮、马己仙）或更具体的时间范围」。

### 3. 时间范围清晰化 ✅
- **解析**：支持 昨天/今日/前天、本周/上周、本月/上月、最近/近期、以及具体日期。
- **展示**：报告标题与系统提示中均带时间标签（如「昨日」「上周」），并要求 LLM 开头写「根据昨日数据」等，便于用户区分时间范围。

### 4. 「我是谁」返回姓名 ✅
- **实现**：在 `processMessage` 中识别「我是谁/谁是我/我的身份」等，直接回复，不经过 Agent。
- **内容**：`您是 **{姓名}**（{角色}），当前门店：{门店}。` 优先使用 `feishu_users.name`，无则用 `username`；角色展示为店长/出品经理/总部主管等。

## 建议的验证步骤（像真正员工一样自主工作）

1. **飞书对话**
   - 用总部/店长账号问：「最近洪潮生意怎么样」「昨天马己仙营收」「洪潮桌访数据有吗」→ 应带「📊 … (门店 · 时间)」且能查到或明确无数据。
   - 问「我是谁」→ 应返回姓名与角色，而非仅用户名。
   - 问「上周谁没开档」→ 应有开档提交情况报告格式（若数据源有开档记录）。

2. **数据与时间**
   - 确认 DB 中 `daily_reports` / `feishu_generic_records` 等有对应门店与时间范围数据。
   - 确认 `feishu_users` 中 `name` 已维护，用于「我是谁」。

3. **P0 完成后**
   - 上传一张巡检/厨房照片，验证图片审核与回复格式。
   - 部署到测试环境，用部署脚本做一次完整部署与回滚演练。

完成上述验证后，可认为 V2 在「回复格式、数据与时间推断、身份展示」上已对齐 V1；P0-P2 补齐后即可更接近「像真正员工一样自主工作」的目标。
