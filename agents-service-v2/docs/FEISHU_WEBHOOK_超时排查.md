# 飞书「请求3秒超时」排查

## 原因说明

飞书在保存「请求地址」时会向该 URL 发 **POST** 做校验。出现 **请求3秒超时** 表示：**飞书服务器在 3 秒内连不上你的服务器**，常见是网络/防火墙问题，而不是业务代码问题。

## 必做：放行 3100 端口

Agents V2 监听 **3100**。需要让公网能访问 `http://你的服务器IP:3100`。

### 阿里云 ECS

1. 登录 **阿里云控制台** → **ECS** → 找到对应实例（如 47.100.96.30）。
2. 进入 **安全组** → **配置规则** → **入方向** → **手动添加**：
   - 端口范围：**3100/3100**
   - 授权对象：**0.0.0.0/0**（或按需缩小为飞书 IP 段）
   - 协议：**TCP**
3. 保存后，在本地用浏览器或 curl 测试：
   ```bash
   curl -s http://47.100.96.30:3100/health
   ```
   若返回 JSON 且含 `"ok":true`，说明端口已通。

### 本机防火墙（若未用安全组或有多层）

- **firewalld**：`sudo firewall-cmd --add-port=3100/tcp --permanent && sudo firewall-cmd --reload`
- **iptables**：放行 3100 入站。

## 飞书侧请求地址

- 保存的地址建议为：`http://47.100.96.30:3100/api/webhook/feishu`
- 若飞书要求 **HTTPS**，则需在服务器前加一层 **Nginx（或其它反向代理）** 配 SSL，请求地址改为：`https://你的域名/api/webhook/feishu`，并在 Nginx 里把 `/api/webhook/feishu` 反向代理到 `http://127.0.0.1:3100`。

## 验证

端口放行后，在飞书开放平台再次点击「保存」，应不再出现「请求3秒超时」。若仍超时，可检查：

- 服务器是否只监听了 127.0.0.1（本应用默认监听 0.0.0.0，无需改）
- 是否有其它安全策略或代理拦截 3100。
