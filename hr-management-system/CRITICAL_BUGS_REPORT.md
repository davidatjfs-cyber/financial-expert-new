### 问题1：责任人跨门店分派错误 🔴

**问题描述**：
洪潮大宁久光店的营销活动任务被错误分派给马己仙上海音乐广场店的出品经理。

**错误数据**：
```
任务门店：洪潮大宁久光店
错误分派：黎永荣（NNYXLYR04）- 马己仙上海音乐广场店出品经理
正确应该：王世波（NNYXWSB39）- 洪潮大宁久光店出品经理
```

**根本原因**：
1. `resolveAssignee` 函数在接收到 `existingAssignee` 参数时，直接使用该值，不验证门店匹配
2. 测试时手动创建任务指定了错误的 `assignee_username`
3. 缺少跨门店分派的校验机制

**影响**：
- ❌ 任务通知发送给错误的人
- ❌ 责任人无法执行任务（不在该门店）
- ❌ 绩效评估计入错误的人员
- ❌ 飞书消息推送到错误的人

**修复方案**：
```javascript
async function resolveAssignee(category, store, existingAssignee) {
  const state = await getSharedState();
  const normalizedStore = normalizeStoreKey(store);
  
  // 如果有指定责任人，验证其是否属于目标门店
  if (existingAssignee) {
    const all = [
      ...(Array.isArray(state?.employees) ? state.employees : []),
      ...(Array.isArray(state?.users) ? state.users : [])
    ];
    const user = all.find(u => u.username === existingAssignee);
    
    // 验证门店匹配
    if (user && normalizeStoreKey(user.store) === normalizedStore) {
      return { 
        username: user.username, 
        name: user.name, 
        role: user.role, 
        store 
      };
    } else if (user) {
      console.warn(`[resolveAssignee] User ${existingAssignee} belongs to ${user.store}, not ${store}. Auto-resolving...`);
      // 继续按门店自动匹配
    }
  }
  
  // 按门店自动匹配逻辑...
}
```
