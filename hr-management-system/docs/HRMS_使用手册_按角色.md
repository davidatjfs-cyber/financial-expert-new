# HRMS 使用手册（按角色）

版本：v1.1（基于当前系统实现）  
适用对象：管理员、总部营运、总部人事、门店店长、出纳、出品经理、门店员工

---

## 1. 手册目标

本手册用于帮助不同角色快速上手 HRMS，并按“真实业务流程”完成日常工作。重点覆盖：

- 登录与基础操作（移动端优先）
- 按角色可见模块与权限边界
- 关键审批流程（入职/离职/请款/晋升/奖惩/休假/积分）
- 报表、工资补贴、积分、日报等操作流程

---

## 2. 系统角色说明

系统内主要角色：

1. 系统管理员（admin）
2. 总部营运（hq_manager）
3. 总部人事（hr_manager）
4. 门店店长（store_manager）
5. 总部出纳（cashier）
6. 门店出品经理（store_production_manager）
7. 门店员工（store_employee）

> 说明：系统支持中文角色名自动映射到标准角色编码，建议组织内部统一使用标准角色含义。
 
> 重要：多数业务数据存在“门店隔离”规则。门店店长/门店员工默认只能看到本店数据；跨店查看通常仅对管理员/总部角色开放。

---

## 3. 通用操作（所有角色）

### 3.1 登录系统

1. 打开系统地址，输入用户名与密码。
2. 登录成功后，左侧导航会自动按角色显示可访问模块。
3. 首页默认为“我的档案”。

提示：

- 移动端使用时，可通过底部导航快速切换模块。
- 若出现“权限不足/看不到菜单”，先确认账号角色是否正确，再联系管理员。

### 3.2 导航与红点提醒（未读红点/数字）

- 导航中的红点数字用于提示“未读/未处理事项”。
- 常见有红点提醒的模块：
  - 待审批
  - 请款
  - 培训/学习任务
  - 考试
  - 奖惩
- 进入模块并查看对应记录后，未读数量会下降。
- 红点属于“提醒机制”，不等同于流程状态；是否需要你处理，以列表中的“待我审批/待我支付/待我完成”等筛选为准。

### 3.3 我的档案（建议每日查看）

可查看：

- 基本信息、门店与岗位
- 当前状态（在职/停用）
- 累积奖励/累积处罚（入职至今累计金额）
- 本月奖励/本月处罚金额（若开启展示）
- 培训与考试记录
- 晋升与审批相关记录（根据角色展示）

## 4. 角色-模块权限速查

> 说明：以下为当前实现下的“默认可访问”能力（最终以系统实际配置为准）。

### 4.1 系统管理员（admin）

- 全量访问：员工管理、待审批、请款、奖惩、积分、日报、分析报表、系统设置
- 可执行关键管理动作：
  - 删除审批单
  - 导出请款数据
  - 工资审核
  - 数据导入导出
  - 角色权限配置

### 4.2 总部营运（hq_manager）

- 可访问：员工管理、待审批、请款、奖惩、日报、分析报表
- 重点职责：跨门店审批、运营类报表查看、请款审批

### 4.3 总部人事（hr_manager）

- 可访问：员工管理、待审批、请款、奖惩、积分、分析报表
- 重点职责：人事审批、奖惩审批、工资补贴调整、积分规则维护

### 4.4 门店店长（store_manager）

- 可访问：员工管理、待审批、请款、奖惩、积分、日报、分析报表
- 重点职责：
  - 发起入职申请（非直接建档）
  - 发起离职/晋升/奖惩/请款
  - 审批下属员工申请
  - 管理本店员工数据

### 4.5 总部出纳（cashier）

- 可访问：待审批、请款（查看/支付）
- 重点职责：按审批结果执行打款并回写支付状态

### 4.6 门店出品经理（store_production_manager）

- 可访问：我的档案、分析报表（查看权限）
- 重点职责：查看业务与出勤相关数据，配合门店管理执行

### 4.7 门店员工（store_employee）

- 可访问：我的档案、培训、考试、积分（申请）
- 重点职责：完成学习考试、提交积分申请、发起个人流程（如休假/离职）

## 5. 核心业务流程（详细步骤）

## 5.0 待审批模块（所有审批处理的统一入口）

适用角色：店长、总部营运、总部人事、管理员、出纳（部分单据）、以及任何被配置为审批人的角色。

待审批常见使用方式：

1. 进入“待审批”。
2. 通过筛选快速定位：
  - 我待审批 / 我已审批
  - 按单据类型：入职、离职、休假、晋升、奖惩、请款、积分等
  - 按门店/月份（如页面提供）
3. 打开单据详情：
  - 核对关键信息（门店、申请人、金额/天数/生效日期）
  - 查看历史审批记录（上一节点意见）
4. 执行操作：通过 / 驳回，并填写审批意见。

提示：

- 审批意见建议包含：结论 + 依据 + 生效时间（若有）。
- 若你发现“审批人不对/流程不符合公司规则”，联系管理员检查审批流配置。

## 5.1 入职流程（门店店长发起）

适用角色：店长（发起）→ 上级审批人 → 人事经理 → 管理员

### 发起步骤（店长）

1. 进入“员工管理”或“待审批”相关入口，点击“新增员工/发起入职”。
2. 填写新员工信息：姓名、账号、门店、岗位、入职日期等。
3. 确认入职日期必填（系统强校验）。
4. 提交申请。

### 审批步骤（审批人）

1. 进入“待审批”。
2. 筛选类型为“入职”。
3. 核对员工信息。
4. 选择“通过”或“驳回”，填写审批意见。

### 生效规则

- 仅当全部审批节点通过后，员工账号/档案才正式进入系统。
- 若驳回，流程终止，需重新发起。

## 5.2 离职流程

适用角色：员工/管理者发起 → 直属上级 → 总部营运 → 人事经理

### 发起步骤

1. 打开“待审批”或离职申请入口。
2. 填写离职日期、离职原因等。
3. 提交后进入多级审批。

### 审批步骤

1. 审批人进入“待审批”。
2. 查看离职申请详情。
3. 可补充离职类型（自愿/非自愿）等信息。
4. 通过后自动流转到下一审批人。

### 生效规则

- 全部通过后，系统按离职日期执行状态变更（账号停用/离职生效）。

## 5.3 休假流程

适用角色：员工发起；管理线审批

### 发起步骤

1. 进入申请入口，选择“休假”。
2. 必填开始日期、结束日期。
3. 提交申请。

### 审批步骤

1. 在“待审批”中处理休假单。
2. 审批人可填写备注；必要时可补充剩余假期天数信息。
3. 逐级流转至终审。

## 5.4 晋升流程

适用角色：店长/总部管理发起，管理线审批

### 发起步骤

1. 进入“晋升申请”页面。
2. 如系统要求走“培训/考核晋升轨道（Promotion Track）”，先在晋升页面完成：
  - 训练/学习环节（按系统提示完成）
  - 考核/测评环节（提交后等待记录入库）
3. 填写目标级别、晋升理由、佐证信息。
4. 提交申请。

### 审批步骤

1. 待审批中筛选“晋升”。
2. 核对绩效、培训考试记录、岗位编制。
3. 若该晋升属于“正式晋升/店长任命”等类型，审批详情中可能包含：
  - 训练/测评完成情况
  - 导师/训练安排（如系统提供配置项）
  - 目标薪资/生效时间
4. 审批通过后自动进入下一节点。

### 建议

- 审批意见中明确“生效时间、薪资调整、岗位变更说明”。

## 5.5 奖惩流程

适用角色：管理员/总部营运/总部人事/店长

### 发起步骤

1. 进入“奖惩管理”。
2. 选择奖惩对象，填写奖惩类型、事由、结果、金额。
3. 提交审批。

### 审批步骤

1. 进入“待审批”，筛选“奖惩”。
2. 审核事实依据与金额合理性。
3. 通过后自动写入奖惩记录，并在档案中可追溯。

### 查询

- 发起人可在“奖惩管理”中查询已完结记录。
- 员工可在“我的档案”查看累计奖励/累计处罚。

## 5.6 请款流程

适用角色：管理员/总部营运/总部人事/店长发起，出纳执行支付

### 发起步骤

1. 进入“请款”页面，点击“新建请款”。
2. 填写门店、日期、金额、请款分类。
3. 提交审批。

### 审批步骤

1. 审批人在“待审批”查看请款单。
2. 可按“我待审批/我发起/全部”筛选（具备权限角色可看全部）。
3. 审批通过后进入可支付状态。

### 支付步骤（出纳/授权角色）

1. 打开请款单详情，核对付款信息。
2. 点击“支付/已支付”。
3. 系统更新状态为 paid，用于报表统计。

### 预算检查建议

- 发起前先查看预算汇总（同门店、同月份、同类目已用金额）。
- 超预算时应要求补充说明并走强化审批。

## 5.7 积分流程

分为“规则维护”“员工申请”“审批落地”三段。

### A. 规则维护（管理员/人事）

1. 进入“员工积分”。
2. 新增积分规则：门店、事项名称、分值。
3. 可启用/停用规则。

### B. 员工申请（门店员工）

1. 进入“员工积分”页面。
2. 选择事项，填写申请原因。
3. 上传证明图片（可多张）。
4. 提交积分申请。

### C. 审批与记录

1. 审批人进入“待审批”处理积分申请。
2. 通过后形成积分记录。
3. 管理层可按门店查询积分记录；店长默认查看本店。

## 5.8 营业日报流程

### 查看权限

- 管理员、总部营运、店长可查看。

### 填报权限

- 默认仅管理员可新增/编辑（按当前实现）。

### 操作步骤

1. 进入“营业日报”。
2. 按日期与门店录入关键经营指标。
3. 保存后可在报表模块联动分析。

## 5.9 分析报表流程

包含：业务报表、离职分析、出勤报表、工资报表、晋升记录、备货预测等（按权限展示）。

### 使用步骤

1. 进入“分析报表”。
2. 选择报表类型与月份/门店。
3. 查看统计结果并导出（管理员具备更完整导出能力）。

#### 晋升记录（管理员/总部人事/总部营运）

用途：汇总“已审批通过的正式晋升”记录，便于跨店追溯审批人、审批时间与目标薪资等关键信息。

建议使用：

- 月度复盘时先看“晋升记录”，再联动“工资报表/薪资变更”核对是否已按期生效。

#### 备货预测（如已上线展示）

用途：基于历史样本 + 输入条件（日期、天气、假日、预计营收等）预测产品销量，用于外卖/堂食、午市/晚市的备货决策。

常见操作：

1. 选择门店（门店店长默认仅本店）。
2. 选择业务类型（外卖/堂食）与时段（午市/晚市）。
3. 填写预测条件并提交预测。
4. 如有“历史样本上传”，可批量导入 CSV/JSON（按系统提示格式），用于提升预测准确率。

### 工资相关重点

- 工资表支持补贴字段。
- 管理员与人事可调整补贴。
- 管理员/总部营运可执行工资审核标记。

## 6. 按角色“每日工作清单”

## 6.1 管理员（admin）

每日建议顺序：

1. 打开“待审批”处理关键单据（入离职/请款/奖惩/晋升/积分/休假）。
2. 进入“员工管理”核对新增/离职状态。
3. 检查“请款”支付链路与预算使用。
4. 在“分析报表”复核工资、出勤、业务指标。
5. 必要时在“设置”调整角色权限、字典配置。

## 6.2 总部营运（hq_manager）

1. 待审批优先处理跨店流程。
2. 检查日报与业务报表异常门店。
3. 对店长提报事项给出审批意见和整改要求。

## 6.3 总部人事（hr_manager）

1. 处理入离职、晋升、奖惩审批。
2. 在工资报表中维护补贴调整。
3. 维护积分规则，复核异常积分申请。

## 6.4 门店店长（store_manager）

1. 处理门店员工审批（尤其入职/离职/休假/积分）。
2. 发起请款并跟踪进度。
3. 维护本店员工档案与状态。
4. 日终查看营业日报与关键报表。

建议：

- 新员工入职后，关注“入职日期（joinDate）”是否正确；员工列表通常可按入职日期排序，便于排班与管理。
- 如出现“发起流程时找不到上级审批人”，需要管理员完善你的上级（managerUsername）配置。

## 6.5 出纳（cashier）

1. 在“待审批/请款”查看已审批通过单据。
2. 执行支付并更新状态。
3. 对账后反馈异常单据给发起人。

## 6.6 出品经理（store_production_manager）
 
1. 查看档案与报表。
2. 关注出勤、绩效相关指标，配合店长执行改进。
 
## 6.7 门店员工（store_employee）
 
你在系统里的高频动作主要是“看自己的信息 + 做任务 + 发起个人申请 + 跟进进度”。下面按使用场景给出详细步骤（入口名称以页面实际显示为准）。
 
### 6.7.1 场景 A：查看我的档案（每日/每周建议）
 
适用：所有门店员工
 
操作步骤：
 
1. 登录后进入“我的档案”。
2. 核对基础信息：姓名、门店、岗位、账号状态（在职/停用）。
3. 查看“累积奖励/累积处罚”“本月奖励/本月处罚”（若开启展示）。
4. 查看培训/考试进度（是否有未完成项）。
5. 若发现信息有误（门店/岗位/入职日期等），联系店长或管理员修改，不要自行重复注册账号。
 
常见问题：
 
- 档案里看不到某些数据：通常是权限或该功能未对员工开放；优先问店长确认公司规则。
 
### 6.7.2 场景 B：完成培训/学习任务
 
适用：所有门店员工
 
操作步骤：
 
1. 进入“培训/学习”模块（移动端可从底部导航进入）。
2. 在列表里优先查看：
  - 待完成
  - 即将到期
3. 打开某个任务进入详情：阅读要求、完成标准、截止时间。
4. 按要求完成学习内容（观看/阅读/打卡等，以系统实际提供为准）。
5. 看到“已完成/已提交/待审核”等状态后，返回列表确认状态变化。
 
进度/结果查看：
 
- 你可在“我的档案”中查看培训完成记录与历史。
 
### 6.7.3 场景 C：参加考试/测验
 
适用：所有门店员工
 
操作步骤：
 
1. 进入“考试”模块。
2. 找到“待考试/进行中”的考试，打开详情确认：考试时长、题量、及格线、是否允许重考。
3. 点击“开始考试”。
4. 按题目逐题作答：
  - 单选/多选：选择选项
  - 判断/填空：按题干填写
5. 考试结束前点击“提交”。
6. 提交后查看结果：分数、是否通过、错题（若系统提供）。
 
注意事项：
 
- 考试过程中尽量不要刷新页面或退出登录，避免答案丢失。
- 若出现网络中断，优先截图保存并联系店长/管理员确认是否需要补考。
 
### 6.7.4 场景 D：积分申请（上传凭证）
 
适用：门店员工发起；店长/管理层审批
 
操作步骤：
 
1. 进入“员工积分”。
2. 在可申请事项里选择对应规则（例如：学习完成、门店改善、优秀表现等）。
3. 填写申请说明：
  - 做了什么
  - 发生时间
  - 可量化结果（如有）
4. 上传证明图片（可多张）。建议：拍清晰、包含关键信息（时间/内容/签字/现场）。
5. 点击“提交申请”。
 
如何查看进度：
 
1. 在积分模块切换到“我的申请/我的记录”（若提供）。
2. 查看状态：
  - 待审批：等待店长/管理层处理
  - 已通过：积分已入账
  - 已驳回：查看驳回原因，补齐材料后重新提交
 
常见问题：
 
- 找不到合适的事项：说明当前门店未配置对应积分规则，联系店长或总部人事。
- 反复驳回：优先补齐“说明 + 凭证”，并确保与规则匹配。
 
### 6.7.5 场景 E：休假申请（请假）
 
适用：门店员工发起；管理线审批
 
操作步骤：
 
1. 进入“待审批/申请”入口，选择“休假”。
2. 选择休假类型（如系统提供：事假/病假/年假等）。
3. 填写开始日期、结束日期（必填）。
4. 填写原因说明；如需附件证明（如病假证明），按要求上传。
5. 点击“提交”。
 
如何跟进：
 
1. 在“待审批”中切换到“我发起”。
2. 打开申请查看审批进度与当前节点。
3. 若被驳回，按意见修改后重新发起。
 
注意事项：
 
- 尽量提前发起，避免影响排班。
- 日期务必准确（跨天、跨月要核对），否则后续出勤/工资统计可能出现偏差。
 
### 6.7.6 场景 F：离职申请（员工主动发起）
 
适用：门店员工发起；管理线审批
 
操作步骤：
 
1. 进入“待审批/申请”入口，选择“离职”。
2. 填写离职日期（必填）与离职原因。
3. 如公司要求交接清单/资产归还证明，可按要求上传附件。
4. 点击“提交”。
 
如何查看结果：
 
1. 在“待审批”切换到“我发起”。
2. 查看审批是否全部通过。
3. 全部通过后，系统会按离职日期执行状态变更（账号停用/离职生效）。
 
### 6.7.7 场景 G：查看我发起的申请进度与结果（通用方法）
 
适用：休假、离职、积分等所有“员工可发起”的流程
 
操作步骤：
 
1. 进入“待审批”。
2. 切换到“我发起”。
3. 按类型筛选（如有）：休假/离职/积分等。
4. 打开单据详情查看：
  - 当前审批节点是谁
  - 历史审批意见
  - 是否需要你补充材料
 
建议：
 
- 若超过公司约定时限仍未处理，先联系你的直属店长/上级确认；不要重复发起多张同类申请。
- 尽量在备注中写清楚关键信息，减少来回沟通。

## 7. 常见问题与排查

### 7.1 看不到某个模块

原因通常是角色权限不足。处理建议：
 

1. 先确认当前账号角色是否正确。
2. 由管理员在权限配置中检查角色权限。
3. 重新登录刷新菜单权限。

### 7.2 审批单无法提交

常见原因：

- 必填字段缺失（如入职缺入职日期、请款缺金额）
- 同类型存在“待审批中”重复申请
- 申请人未配置上级（managerUsername）

### 7.3 通过后未生效

处理建议：

1. 确认审批链是否“全部通过”。
2. 检查是否设置了生效日期（如离职日期）。
3. 管理员在报表/档案中核验最终状态。

## 8. 管理建议（落地）

1. 统一组织岗位命名，避免同义角色混用。
2. 审批意见必须写结论+依据+生效时间。
3. 请款务必与预算联动，超预算走补充说明。
4. 人事每周检查离职、晋升、奖惩闭环数据。
5. 管理员每月导出关键报表，形成经营与人效复盘。

## 9. 附录：关键流程总览（便于培训）

- 入职：店长发起 → 上级 → 人事 → 管理员 → 生效入库
- 离职：发起人 → 上级 → 总部营运 → 人事 → 到期生效停用
- 请款：发起 → 审批链 → 出纳支付 → 进入财务统计
- 奖惩：发起 → 审批 → 写入奖惩记录与员工档案
- 积分：员工申请 → 审批 → 写入积分记录
