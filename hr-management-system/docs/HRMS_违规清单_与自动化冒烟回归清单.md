# HRMS 违规清单（基于近期重复问题沉淀）

> 目标：把“反复踩坑”变成“不可再犯”的硬约束。

## A. 发布与入口类违规（高频）

1. **只改了非线上入口文件**
   - 现象：本地看到变更，线上无变化。
   - 违规判定：修改未覆盖实际入口页面（例如只改 `working-fixed.html` 或只改 `mobile-nav-production.html` 其中之一）。

2. **前端变更后未 bump SW cache**
   - 现象：用户端持续命中旧缓存。
   - 违规判定：前端页面改动后，`sw.js` 的 `CACHE_NAME` 未递增。

3. **发布后未做版本一致性核对**
   - 现象：文件已传但服务/缓存仍旧版本。
   - 违规判定：未核对服务器文件时间戳 + 未做健康检查 + 未确认页面关键片段已上线。

4. **错误路径发布**
   - 现象：rsync/scp 成功但逻辑没变。
   - 违规判定：目标路径不是 `/opt/hrms/hr-management-system/...` 对应真实运行路径。

## B. 功能与交互类违规（高频）

5. **移动端弹窗交互不符合人体工学**
   - 现象：弹窗出现在顶部、遮挡严重、难以单手操作。
   - 违规判定：移动端未优先底部抽屉（或至少靠近触发按钮）。

6. **角色权限回归**
   - 现象：店长/出品经理/总部权限串位，出现 403 或越权。
   - 违规判定：修改后未覆盖角色矩阵回归测试。

7. **日报上传/提交 403 回归**
   - 现象：日报图片上传失败、提交失败。
   - 违规判定：发布后未验证 `/api/uploads/daily-report` 与日报提交流程。

8. **通知中心视觉改版未全量同步**
   - 现象：部分入口仍是旧“公司通知”样式。
   - 违规判定：通知 UI 仅在单文件修改，未同步到实际入口文件。

## C. 数据与安全类违规（红线）

9. **覆盖 `server/uploads/` 目录**（红线）
   - 违规判定：任何全量同步未排除 `server/uploads/`。
   - 后果：用户文件永久丢失风险。

10. **无回滚预案发布**
   - 违规判定：发布前无备份、无回滚指令。

# 自动化冒烟测试回归清单（可执行）

> 目标：发布后 5~10 分钟内自动确认“核心链路没坏”。

## 1) 发布后基础健康检查（必须）

```bash
# 服务器
systemctl is-active hrms.service
curl -s http://127.0.0.1:3000/api/health
```

通过标准：
- 服务状态 `active`
- `health` 返回 `{"ok":true,...}`

## 2) 版本与文件一致性检查（必须）

```bash
# 服务器
ls -l /opt/hrms/hr-management-system/working-fixed.html \
      /opt/hrms/hr-management-system/mobile-nav-production.html \
      /opt/hrms/hr-management-system/sw.js

grep -n "CACHE_NAME" /opt/hrms/hr-management-system/sw.js
```

通过标准：
- 三个文件时间戳为本次发布时间
- `CACHE_NAME` 为本次新版本号

## 3) 通知中心回归（前端）

检查点：
- 出现新通知头部元素：`#profile-notif-section`
- 脉冲点元素存在：`#profile-notif-pulse`
- 通知可展开/收起：`#profile-notif-toggle`

建议自动化（Playwright/Cypress）断言：
- 登录后进入 profile 页，`#profile-notif-section` 可见
- 点击“查看更多通知”后，`#profile-notif-more` 带 `expanded` 类

## 4) 日报“添加人员”弹窗定位回归（前端）

检查点：
- 点击“添加”后 `#dr-staff-modal .modal-content` 的 `top` 应接近屏幕底部（移动端）
- 不应出现 `top < 80px` 的顶部弹窗

建议自动化断言（移动视口）：
- 视口 iPhone 12
- 打开日报编辑页点击“添加”
- 获取 modal-content bounding box：
  - `y > viewportHeight * 0.2`
  - `y + height <= viewportHeight`

## 5) 权限与403回归（接口）

### 5.1 日报上传接口
- 用店长账号上传日报图片应成功（非 403）

### 5.2 日报提交接口
- 店长可提交
- 非授权角色被拒绝（符合权限矩阵）

通过标准：
- 授权角色不报 403
- 非授权角色按预期拒绝

## 6) 高风险业务回归（最小集）

1. 日报：新增前厅/厨房/休息/排班人员并保存
2. 公司通知：至少渲染 1 条，时间与样式正常
3. 上传：日报附件上传成功
4. 首页/个人页无报错（控制台无红错）

# 建议接入 CI 的“发布门禁”

## Gate-1（阻断级）
- 服务健康检查失败 -> 阻断
- 线上文件时间戳未更新 -> 阻断
- SW cache 未 bump（检测到前端 diff）-> 阻断

## Gate-2（警告级）
- 控制台出现严重错误 -> 警告
- UI 关键节点缺失（通知区、日报弹窗）-> 警告
