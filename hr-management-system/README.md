# 企业人力资源管理系统 v2.0.3

## 🏢 系统概述

企业人力资源管理系统是一个集成知识库管理、员工考试测评、晋升路径管理、培训学习跟踪和奖惩管理的企业人力资源综合管理平台，帮助企业提升员工管理和人才培养效率。

## 🎯 核心功能

### 👥 角色权限管理
- **管理员**: 拥有所有功能的最高管理权限
- **总部管理层**: 可查看公司所有员工信息和培训资料，可对店长进行奖惩
- **门店店长**: 可查看自己门店的员工信息和自己的培训记录，可对门店员工进行奖惩
- **总部员工**: 只能查看自己的档案和培训记录
- **门店员工**: 只能查看自己的档案和培训记录

### 📚 知识库管理系统
- 知识库文档上传、分类和管理
- 类似Google Notebook的功能，支持知识整理和分享
- 文档权限设置，根据不同部门或角色开放访问权限
- 文档搜索和标签管理功能

### 🎓 培训学习管理系统
- 培训资料界面：展示各类培训文档和课程资源
- 本月累计学习时间统计：显示登录者本月在培训资料界面的总学习时长
- 学习时间计算逻辑：进入培训资料预览界面开始计时，退出预览界面停止计时
- 支持学习进度暂停和继续功能
- 培训记录管理：记录员工的学习历史和完成情况

### 📝 考试测评系统
- 考题上传和题库管理
- 按门店、岗位或特定员工分发考试
- 自动评分和成绩统计
- 考试结果分析和报告生成
- 员工学习进度跟踪
- 考试界面设计：采用简洁的卡片式布局，顶部显示HR管理系统标题，中间显示考试内容区域，底部设置答题和提交按钮，界面配色以白色和蓝色为主

### 📊 题库管理系统（管理员首页功能）
- 管理员可在首页快速访问题库管理模块
- 题库分类管理：按科目、难度、部门等维度分类
- 题目批量导入导出功能
- 题目搜索和筛选功能
- 题目编辑和审核流程
- 题库统计分析：题目数量、更新时间、使用频率等数据展示

### 📈 员工晋升管理系统
- 晋升路径设定和配置
- 员工绩效评估指标设置
- 学习成果和工作表现综合评价
- 晋升资格自动判定
- 晋升审批流程管理
- 淘汰机制设置和通知管理

#### 🔄 晋升路线分类
- **同岗位晋升**: 员工在当前岗位上的技能提升和熟练度增长，成为熟练工
- **跨岗位晋升**: 员工从当前岗位转向其他岗位的发展路径

#### 🏪 前厅人员晋升路线
服务员 → 2. 水吧 → 3. 主管 → 4. 档口 → 5. 经理 → 6. 店长

#### 👨‍🍳 后厨人员晋升路线
打荷 → 2. 档口 → 3. 切配 → 4. 烧味 → 5. 炒锅 → 6. 厨师长

### 📁 员工档案管理系统
- 员工基础信息录入和维护
- 学习记录和考试历史查看
- 职业发展轨迹追踪
- 工资调整和职位变更记录
- 档案模板导入功能：支持按照标准模板批量导入员工档案信息
- 档案导出功能：支持将员工档案导出为标准格式文件
- 管理员专属权限：只有管理员拥有新建档案、编辑档案和删除档案的权限
- 员工档案查看功能：各层级用户可根据权限查看相应的员工档案信息

#### 📋 员工档案详细字段
- **基本信息**: 姓名、性别、出生日期、籍贯、手机号
- **就职信息**: 门店、岗位、级别、入职时间、入职渠道
- **薪资信息**: 现在工资、调薪记录
- **个人能力**: 工作特长、工作成绩
- **绩效记录**: 公司表扬记录、公司批评处罚记录
- **职业发展**: 考试记录、晋升记录、上级评价记录、培训学习记录
- **奖惩记录**: 显示员工的所有奖惩历史记录

### 🏪 门店管理
- 门店信息录入和维护
- 门店员工管理
- 门店培训记录管理
- 门店信息编辑功能：支持门店名称、地址、负责人、联系电话、营业时间等信息的新增、修改和删除操作

### 👤 系统用户管理
- 用户注册和认证
- 角色分配和权限设置
- 用户状态管理
- 用户信息编辑功能：支持用户名、真实姓名、所属部门、岗位、权限角色等信息的新增、修改和删除操作

### 📢 公告管理
- 公告发布和管理
- 公告推送通知
- 公告查看统计

### 📱 线下输入功能
- 支持在无网络环境下录入员工信息和数据
- 离线模式下保存输入内容
- 网络恢复后自动同步线下输入的数据
- 支持手动触发同步功能
- 离线状态提醒和数据缓存管理

### 🏆 奖惩管理系统
- 奖惩模版管理：管理员可以创建和管理各种奖惩模版
- 模版字段：奖惩人、门店、岗位、奖惩事由、奖惩结果、提报人
- 奖惩分发机制：
  - 管理员可以将奖惩模版分发给门店，店长可以开给门店员工，总部管理层可以开给店长
  - 支持精准发布给具体个人：在奖惩发放界面可以选择具体员工作为奖惩对象
  - 支持批量发布：可选择多个员工进行批量奖惩操作
- 发布记录追踪：记录每次奖惩发放的详细信息和状态
- 奖惩记录：所有的奖惩记录会自动记录到员工档案中
- 奖惩流程：提报、审批、下发、记录

- 考试界面采用居中对齐的卡片式设计

## 📊 数据库设计

系统包含以下主要数据表：
- users (用户表)
- employees (员工档案表)
- stores (门店表)
- knowledge_base (知识库表)
- training_records (培训记录表)
- exam_questions (考试题目表)
- exam_results (考试结果表)
- promotion_records (晋升记录表)
- reward_punishment (奖惩记录表)
- announcements (公告表)

## 🔐 权限控制

系统采用基于角色的访问控制(RBAC)：
- 每个角色对应不同的功能权限
- 数据访问权限按角色和门店进行隔离
- 敏感操作需要相应权限验证

## 📱 响应式设计

- 支持桌面端、平板和手机端访问
- 自适应布局确保在不同设备上的最佳体验
- 触摸友好的交互设计
